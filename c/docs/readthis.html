Atomic & Safe Writes

Writes to a temporary file first, then atomically renames using rename(). No partial or corrupted files on crash.

Streaming & Efficient

Processes files in small chunks. Works with very large files without loading everything into memory.

Automatic Decryption

The wrap mode is stored inside the encrypted file. Just run otp D file.enc — no flags needed.

Usage

Simple, memorable commands.

ENCRYPT
$ otp E secret.txt
Encrypted successfully: secret.txt → secret.enc
$ otp E --wrap largefile.dat
Encrypted successfully: largefile.dat → largefile.enc
DECRYPT
$ otp D secret.enc
Decrypted successfully: secret.enc → secret.txt
Note: Decryption is fully automatic. The tool reads the encryption mode from the file header.

Data Safety & Integrity

This tool was designed with reliability as the top priority.

Atomic File Operations

All writes go through a temporary file first. The final file only appears after the data is fully written and synced to disk.

No In-Place Overwrites

The original file is never modified. A new .enc file is always created, giving you a safe backup.

Streaming Processing

Files are processed in small chunks. Even very large files (tens of GB) can be encrypted without high memory usage.

Self-Describing Files

The encryption mode (wrap or not) and original filename are stored inside the .enc file. Decryption requires no extra flags.

Building from Source

Requirements

  • Rust 1.96 or newer
  • Cargo (comes with Rust)

Build Commands

git clone <repo>
cd otp
cargo build --release

The binary will be at target/release/otp

Built with safety and simplicity in mind. No external dependencies.

Key file must be named key.key and placed next to the executable.
